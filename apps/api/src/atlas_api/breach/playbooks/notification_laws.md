# Breach-notification obligations (organizations)

*General information, not legal advice — confirm with counsel. Deadlines and
thresholds change; treat this as a maintained rules table, not gospel.*

| Regime | Who must be notified | Typical deadline |
|---|---|---|
| **US state laws** (all 50 + DC) | Affected residents; often the state AG over a threshold (250–1,000+) | "Without unreasonable delay"; ~20 states set a hard **30–60 days**. The shortest applicable deadline governs. |
| **GDPR** (EU/UK) | Supervisory authority; individuals if high risk | **72 hours** from awareness (authority) |
| **HIPAA** (US health) | Individuals; HHS; media for 500+ in a state | **≤60 days** |
| **CCPA/CPRA** (California) | Private right of action for unencrypted PI; statutory damages $100–$750/consumer | Notification per CA breach statute (~30 days) |
| **GLBA / FTC Safeguards** (US financial) | FTC for 500+ consumers, unencrypted | **≤30 days** |
| **PCI-DSS** (card data) | Card brands / acquirer (contractual, not law) | Per card-brand rules; may require a PCI Forensic Investigator |

Sources: IAPP state chart — https://iapp.org/resources/article/state-data-breach-notification-chart ·
GDPR Art. 33 — https://gdpr-info.eu/art-33-gdpr/ ·
HIPAA Breach Notification Rule — https://www.hhs.gov/hipaa/for-professionals/breach-notification/index.html ·
