01

You describe what leaked.

You type something like “my passwords showed up in a data breach”, tap the data types that were exposed, and hit run. Your browser packages it up and sends it off.

Browser → POST /api/research

02

The nearest city catches it.

Your situation lands at a Cloudflare Worker in the data center closest to you — one of 300+ worldwide. No far-away server to wait on.

Cloudflare edge · runs in milliseconds, near the user

03

Claude triages your situation.

The Worker hands your situation to Claude along with instructions: stay calm, prioritize, cite official sources, keep it plain.

Worker → Anthropic API · claude-opus-4-8

04

It looks up the official guidance.

Instead of guessing, Claude searches — but only the official shelf: the FTC, CISA, the credit bureaus, and other authorities. It writes its own queries.

Native web_search · allow-listed domains · up to 5 searches

05

It assesses urgency.

Real guidance comes back. Claude sorts what to do now from what can wait, and tracks which official source backs each step.

Official results · each becomes a numbered citation

06

Your plan streams back.

Every step flows back the moment it's written — Claude → Worker → your browser — prioritized into do now, do soon, and keep doing.

Server-Sent Events (SSE) · step-by-step

07

You act on it.

You watch the pipeline light up and a clear checklist assemble itself — each step footnoted to the official source you can verify and follow.

Rendered safely · model output escaped, then formatted

08

It's saved for next time.

Your finished plan and its sources drop into the folder, so you can reopen any past plan from your history and keep working through it.

Under the hood

The production edition adds eight more

The live demo runs on Cloudflare. The full production edition runs the same idea on a real cloud — and that's where the rest of the stack lives.

🐍

Python · FastAPI

The kitchen that takes every order.

The async backend service — built from scratch, fully tested.

🤖

LangGraph agents

A team of specialists, each with one job.

Multi-agent graph: plan → search → verify → write, driven by Claude.

🐘

PostgreSQL

The system of record.

Stores users, runs, sources, and reports — the production database.

⚡

Redis + arq workers

The ticket rail and the line cooks.

Queues each breach-response job; workers run it and stream progress back.

☸️

Kubernetes (EKS)

A building that adds rooms on demand.

Runs the API and workers; KEDA autoscales them on the queue.

☁️

AWS

The land the building sits on.

EKS, RDS Postgres, ElastiCache, S3, ECR — owned end to end.

🌍

Terraform

The blueprint that builds the land.

The entire cloud defined as code, per environment.

🚀

GitHub Actions

The crew that tests and ships.

CI/CD — lint, type-check, test, scan, build, and deploy on every change.
