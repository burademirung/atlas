plan.md
01

You describe what leaked.

In the app — a React + Vite single-page app — you type something like “my passwords showed up in a data breach”, tap the data types exposed, and hit run.

React · Vite SPA → POST /api/research

02

It reaches the front door.

It lands at the edge — a Cloudflare Worker, or a FastAPI service in production — but first it's screened: rate limits, a bot check, and allow-listing before anything runs.

Cloudflare Worker / FastAPI · Turnstile · rate limit

03

It joins the queue.

Rather than make you wait on the wire, FastAPI drops the job on a Redis queue and hands your browser a run id to stream from. The slow work happens in the background.

Redis + arq · durable background job queue

04

A worker picks it up.

An arq worker pulls the job and runs a LangGraph agent graph — plan → search → verify → write — with parallel fan-out across sub-questions.

LangGraph · multi-agent, Claude-driven

05

Claude triages your situation.

The graph hands your situation to Claude with clear instructions, plus breach tools — the same ones exposed over MCP, so any agent can call them.

Anthropic API · claude-opus-4-8 · MCP tools

06

It looks up official guidance.

Instead of guessing, Claude searches — but only the official shelf: the FTC, CISA, the credit bureaus, and other authorities. It writes its own queries.

Native web_search · allow-listed domains · up to 5 searches

07

Your plan streams back.

Every step flows back the moment it's written, over Server-Sent Events — Claude → the edge → your browser — prioritized into do now, do soon, keep doing.

Server-Sent Events (SSE) · token-by-token

08

It's saved for next time.

Your finished plan and its sources are written down so you can reopen any past plan and keep working. The live edition uses Cloudflare D1; production uses PostgreSQL.

Cloudflare D1 · or PostgreSQL — runs + sources

09

It's packaged to run anywhere.

The API and the agent workers ship as container images — multi-stage, non-root, minimal base — so what runs in production is exactly what was tested.

Docker · OCI images · non-root

10

It scales itself.

Those containers run on Kubernetes. When the queue backs up, KEDA adds worker pods; when it's quiet, it scales them back toward zero.

Kubernetes (EKS) · KEDA queue autoscaling

11

It all runs on AWS.

The cluster and everything it talks to — managed Postgres, Redis, object storage, the image registry — live on AWS, owned end to end.

AWS · EKS · RDS · ElastiCache · S3 · ECR

12

The cloud is built from code.

None of it is clicked together by hand. Terraform defines every network, cluster, and database as code, reproducibly, one stack per environment.

Terraform · infrastructure as code · remote state

13

It's tested, then shipped.

On every push, the full test suite runs — pytest, Vitest, testcontainers — then GitHub Actions builds the image and deploys, authenticating to AWS over OIDC with no stored keys.

Testing · GitHub Actions · CI/CD via OIDC

14

And watched in production.

Once live, every run is traced and measured — so latency, errors, and token cost are visible, not guessed. OpenTelemetry exports it all to your dashboards.

OpenTelemetry · GenAI semconv · Prometheus

Rigor, not vibes

Built to standard

The principles this system is designed and implemented against — and the published standards behind them, each linked so you can check the source yourself.

Best practices — design & implementation

Engineering
  • Test-driven development — deterministic agent tests with fake models
  • Strict static typing — mypy --strict, TypeScript, Pydantic
  • Infrastructure as Code — every environment reproducible from Terraform
  • CI/CD with SAST + SCA — CodeQL, lint, type-check, dependency scans per push
  • Keyless cloud auth — GitHub OIDC, no long-lived secrets
  • Twelve-factor config — config from env, stateless processes
  • Idempotency, retries with backoff, per-run cost guardrails
Security & AI safety
  • Defense in depth — edge rate limiting, Turnstile, allow-listed search domains
  • Least privilege — scoped IAM, non-root, read-only container filesystems
  • Prompt-injection spotlighting — untrusted sources fenced, never obeyed
  • Grounded generation — entailment-verified, cite-or-don't-claim
  • Data minimization — k-anonymity password check, no sensitive PII retained
  • Secrets management — Wrangler secrets / AWS Secrets Manager
  • Graceful degradation — falls back to known official guidance on tool limits
Design & frontend
  • Accessibility — semantic HTML, visible focus, prefers-reduced-motion honored
  • Progressive enhancement — content streams in via SSE, usable as it loads
  • Design tokens + a single type scale — one consistent visual language
  • Responsive — fluid down to small screens
  • XSS-safe rendering — model output escaped before markdown
  • Fingerprinted assets — versioned URLs so updates never serve stale
